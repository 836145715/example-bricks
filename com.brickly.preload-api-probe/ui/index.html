纯 UI 工具：验证 bricklyPreload.exposeApi 调用是否进入日志中心（action = preload · …）。请在主窗口打开「日志中心 / Trace」对照。

怎么验：点下方按钮 → 主窗口日志中心搜 preload · 或本 brickId。
应有 Trace：成功/失败的门面方法调用。 不应有 Trace：preload 加载时静默写临时文件（旁路 Node）。
preloadProbe: … legacy: … window.brickly: …

1. 成功调用

应产生 success Trace：preload · preloadProbe.echo / getMeta / nested…
期望：每点一次 ≈ 一条 Trace，status=success

2. 调用错误

应产生 failed Trace，带 error.message
期望：页面 catch 到错误，日志中心 status=failed

3. 敏感入参

args 含 token/password，摘要应脱敏为 [redacted]
期望：Trace input.args 中 token 不出现明文

4. 兼容 / 劫持路径

preload 里用旧 API contextBridge.exposeInMainWorld 暴露的方法
期望：仍有 Trace；事件字段 source=contextBridge

5. 绕过 / 旁路观察

这些操作本身不会「再记一条调用」，用于读 getMeta.bypassNotes
期望：加载期 silent fs 无 preload Trace；保留名占用被拒；二次 expose 以最后一次为准

平台 API 不走用户门面记账；用于确认窗口身份
期望：有结果；日志中心不应因此多出 preload · brickly.* 用户门面记录
